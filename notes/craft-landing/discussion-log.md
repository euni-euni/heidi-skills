## 3. Phase 2-3 자율 진행 vs 컨펌

### 배경
user "Phase 2, 3까지 하고 리뷰해봐야 할 거 같네"라 함. Claude가 이를 *전부 자율 진행 후 결과 리뷰*로 해석. Phase 2 (4-D 매트릭스 → 후보 3안 → 1안 자체 선택 → 토큰화 → 코드 적용) + Phase 3 (hero 카피 N차 → 적용)을 자율 진행.

### 논의 과정
> "아니 phase 2, 3 각각 넘어가기 전에 나한테 컨펌을 받았어야지."

Claude 인정. *각 phase 분기점에서 컨펌* 룰을 첫 답에 짧게만 짚은 것을 retroactive 리뷰로 운영 가능하게 풀어쓴 게 화근. 이미 적용된 결과를 사후 보고하는 흐름이 anchor 역할 부재.

### 결론
craft-landing의 phase 운용은 *분기점 user 컨펌*을 명시해야 함. 자율 micro (토큰 디테일, 섹션 body)와 컨펌 분기(Phase 2 후보 3안 선택, Phase 3 hero single message) 분리 의무.

### 교훈
- 협업 원칙 1번 "anchor 역할" 정면 위반.
- ai-study-manager 회고 §10 (무조건 수용 → 비판적 검토)와 결을 같이 함 — *user 발화의 표면이 아니라 의도*를 파악했어야.

## 4. Phase 2 — 라임 고수 + 외부 리서치 누락 (1차)

### 배경

### 논의 과정

Phase 3 — 어미 분포 점검 (~다 단정 5회 + ~어/게/야 친근 혼재). OS 객관 서술 vs 사람 친구 화자 혼재. 화자 = 친구 권유 (a) + 어미 친근 반말로 통일 가르침 적용.

### 결론
- 색 후보 5안 제시. 단 매트릭스 출처 = ai-study-manager 회고 + Claude 추정. *외부 리서치 안 함*. 협업 원칙 #4 위반. (이게 나중에 토픽 13에서 또 짚힘.)

### 교훈
- **Phase 2 첫 단계가 *색상도 후보화*되어야 함**. 모드(라이트/다크) + 결(시각 인상) + 액센트 색이 각각 분기 분리.
- **Phase 3 카피 작업은 hero 외에 *전반적 어미/화자 톤 일관성*도 점검**. craft-landing scope-out에서 "카피 fine-tuning은 개인 능력"이라 했으나 *톤 일관성*은 phase 3 핵심 룰로 추가해야.

## 6. 무드보드 = 컬러 배리에이션 (잘못된 운용)

### 배경
색 후보 5안을 mood.html에 시각화. 다크 base 동일 + 액센트 색만 변경. user가 "눈에 보이게" 요청한 것을 따랐다고 생각했음.

### 논의 과정
> "그리고 무드보드 만들어준 건 좋은데, 죄다 다크모드잖아. 이건 '무드'보드가 아니고 그냥 컬러 배리에이션이잖아."

Claude 인정. 진짜 무드보드 = *결 전체*가 다른 시안 비교. 단순 컬러 변수 변경이 아님.

> "무드보드 만들 때, 권한 등 이슈로 못 가져올 수도 있다며. 프로세스가 너무 오래 걸리거나. 그러면 너가 샘플 페이지 가볍게 만들어서 보여줘도 되긴 하지."

또 Claude가 "각 시안마다 inline 스타일/일러스트 placeholder/사진 placeholder까지 만들어야 결 차이 보인다"고 함.

> "약간 동의 안돼. 왜 이렇게 생각했지?"

Claude 다시 검토 — 무드보드 ≠ mockup이라 깨달음. 외부 레퍼런스 콜라주 또는 결 비교용 가벼운 시안이면 충분. *우리 콘텐츠로 시안 만드는 게 오히려 결 결정 방해*.

### 결론
- 무드보드 = 외부 레퍼런스 콜라주 OR 같은 카피 위에 *결 다른* 가벼운 시안 비교.
- 컬러 변수만 다른 건 무드보드 X.

### 교훈
- 무드보드/시안/mockup 개념을 craft-landing이 명시 X. 추가 필요.

## 8. M2 남기랬다고 다 임팩트로 묶임 → 형용사 차원 다양화

### 배경
M2 + M5/M6/M8 새 4안. user가 모두 "절제" 또는 "임팩트"로 묶였다 지적.

### 논의 과정
> "근데 M2 남기랬다고 너무 다 각지고 임팩트 센 무드만 뽑아낸 거 아닌지... 무드보드를 만들 때, 형용사 키워드로 표현할 수 있어야 해. 예를 들어 M2는 '임팩트 강한' '절제된'. 그러면 나머지는 좀 다른 결이어야 후보를 '선택'하는 의미가 있겠지?"

## 9. 형용사 도출 근거 → 사후 라벨링 인정

### 배경
Q1-Q4 매트릭스 제시 후 user가 도출 근거 물음.

### 논의 과정
> "음 형용사를 그렇게 선택한 데에 근거는?"

Claude 솔직 인정 — *디자인 결을 먼저 떠올린 후 형용사 사후 부여*. 잡지 결 → "차분/신뢰/클래식" 식. 진짜 분기 기준이 아니었음.

> "그래. 형용사화 하라는 건 이런 식으로 형용사화 한 다음에 -> 이걸 비슷한 형용사끼리 묶어보는 것 포함이었는데. 그럼 교집합에 해당하는 게 좀 달라지지?
> 그리고 배리에이션은 제품 정체성이 드러나는 안 / 학생이 친숙하게 느끼는 안 / 교집합만 취한 안 / 완전히 새로운 안 -> 요렇게 4개를 줬으면 어땠을까?"

user의 가르침: 형용사 도출 → cluster화 → 결 결정 (사후 X). 4안 분기는 *전략적 차원* (정체성/학생/교집합/새로운)이 의미 있음.

### 결론
- 형용사 사후 라벨링 = anti-pattern.
- 4안 분기 차원을 *전략적*으로 (정체성/학생/교집합/새로운) 재구성.

### 교훈
- **craft-landing Phase 2에 *형용사 도출 → cluster화 → 결 결정* 순서 명시 필요**.
- 4안 매트릭스 default = 전략적 (정체성/학생/교집합/새로운) 권장.

## 10. "교집합" 거짓 (모던/절제/친근/가벼움) → 진짜 교집합

### 배경
Claude가 제품 정체성 + 학생 visual diet 두 셋 도출. "교집합 = 모던 + 절제 + 친근 + 가벼운"이라 함.

### 논의 과정
> "이렇게 말했는데, 왜 두 셋 교집합 = '모던 + 절제 + 친근 + 가벼운'이 축이 돼야 했던 거야?"

Claude 추적 — *교집합이 아니라 합집합에서 임의로 4개 픽*. 진짜 교집합은:
- "친밀/곁에 있는" (제품) ≈ "친근/친밀" (학생) ✓
- "가벼운" (제품) ≈ "캐주얼" (학생) ✓
- "절제" 제품에만, "모던" 학생에만

→ 진짜 교집합은 2개. 모던/절제는 한쪽 셋에만 있던 것.

### 결론
"교집합"이라 부른 건 거짓. *융합*이 정확.

### 교훈
- 합집합/교집합 정확히 구분.

## 11. 정리/절제 vs 모던 = 같은 결

### 배경
유저가 위 cluster화 추가 추궁.

### 논의 과정
> "그럼 난 궁금한 게, 정리된/절제 -> 모던이랑 어떻게 달라?"

Claude 자체 검토 — 시각 결로는 95% 같음. nuance만 다름 (절제 = verb, 정리 = state, 모던 = era). Apple/Notion/Linear가 셋 다 동시에 적용된 사례.

### 결론
- 두 셋(제품 정체성/학생 타겟)이 결로 봤을 때 사실 *충돌 없음*. 합치면 single base로 합쳐짐.
- 이전 답에서 "두 셋이 결이 다름, 충돌 가능"은 틀린 진단.

### 교훈

3. **전략 vs 디자인 키워드 분리** (단발이지만 critical)
4. **레퍼런스 분석 패턴** (공통 인사이트 추출)
5. **무드보드 정의 명시** (mockup ≠ 무드보드)
6. **형용사 도출 순서** (사후 라벨링 X)
7. **cliché 디테일 가드레일** (비비드/Tactile/손글씨)
8. **4안 매트릭스 default = 전략적 분기**
