# Anti-patterns — Claude 반복 실수 13종

ai-study-manager + edu-ai-os 세션에서 반복 등장한 Claude 실수. 매 phase 시작 시 self-check.

## 1. Selection bias — 익숙한 레퍼런스만 보기

**증상**: 학생 타겟 랜딩인데 Anthropic / Granola / Brilliant 같은 *SaaS B2B 레퍼런스*를 가져옴.

**원인**: Claude가 자기 익숙한(=학습 데이터에서 자주 본) 사이트에 무의식적으로 끌림.

**가드레일**: Phase 2 시작 시 4-D 매트릭스의 **"타겟 visual diet" 슬롯을 가장 먼저 채운다**. 디자이너 본인이 좋아하는 사이트 ≠ 타겟이 일상에서 보는 사이트.

→ 한국 10대 → 인스타/유튜브/틱톡/카카오톡/콴다 (와이즈앱 데이터). 학부모 → 네이버 카페/맘카페 등.

## 2. 무조건 수용 — user 직감 질문에 즉시 동조

**증상**: user의 "이거 별로지 않아?" / "이렇게 하는 게 낫지 않아?" 질문에 즉각 수정.

**원인**: user 의도 = 탐색이지 결정 아님. 그런데 Claude는 동의로 해석.

**가드레일**: 질문 받으면 한 발 물러서 *원안의 근거가 무효화되는지* 독립 재검증. 3개 옵션 + 데이터로 anchor. user가 결정하면 그때 수정.

## 3. 과잉 해석 — 피드백을 행동으로 점프

**증상**: user "숫자가 약한데?" → Claude가 "섹션 삭제"로 점프.

**원인**: 피드백 → 의도 확인 → 액션 단계에서 **의도 확인 단계 누락**.

**가드레일**: 피드백 받으면 "어떤 방향이 좋을까?" 또는 옵션 제시. 즉각 큰 액션 X.

## 4. 배치 편의주의 — 카피 살리려고 구조 양보

**증상**: user 카피 ("뭘 빼야 하는지 알 뿐이야")가 좋아서 그 카피에 맞춰 섹션 구조를 양보.

**원인**: 카피 매력성 ↔ 제품 정합성 분리 못 함. 두 코어 균형이 PRD 명시인데 카피가 한 축으로만 흘러도 그냥 둠.

**가드레일**: **구조 정정이 우선, 카피는 구조에 맞춰 수정**. 매 카피 결정 시 "양 축 다 시그널 됐나?" 점검.

## 5. 추측 사실화 — 데이터 없이 일반론 주장

**증상**: "이게 best practice", "전환율에 유리", "관행" 같은 정당화를 데이터 없이.

**원인**: 디자인 결정 이유를 댈 때 외부 리서치 누락 → 일반론으로 메움.

**가드레일**: 색상/타이포/모티프/레이아웃 매 결정에 **4-D 레퍼런스 매트릭스 중 무엇이 받치는지 명시**. "현대적", "트렌디"는 정당화가 아니다.

## 6. 추측 단정 — 출처 없이 사실처럼 주장

**증상**: "OpenAI/Anthropic/Vercel 다 보라-파랑 dominant" — 거짓. 실제: Anthropic=코랄(#cc785c), OpenAI=그린(#10A37F), Vercel=흑백, Linear만 보라.

**원인**: 추측을 권위 톤으로 단정. 검증 비용 회피.

**가드레일**: 출처 없으면 "추정"이라 표시. "X is Y" 단정은 출처 명시. 의심 들면 WebSearch.

→ 협업 원칙 #2 (검증 우선·확정 서술 금지) 직접 위반.

## 7. 외부 리서치 의무 분기점 — 회피 금지 (반복 가드레일)

**증상**: 학생 visual diet / 트렌드 / 경쟁사 정보를 *내 지식·회고·추정*으로 채움. WebSearch 안 함. **이 패턴 한 세션에 3번 등장** = 가장 잦은 실수.

**원인**: 외부 리서치 = 시간 비용 + 도구 로드. 회피 욕망.

**가드레일**: Phase 2 진입 전 self-check — *이 슬롯에 외부 검색 데이터가 들어갔나?*

- ❌ "ai-study-manager 회고에서 토스/무신사…" — 회고 ≠ 데이터
- ❌ "내 일반 지식으로 라인/인스타…" — 추정
- ✅ "와이즈앱 2025 모바일 앱 사용 분석: 인스타 MAU 412만"

안 했으면 진행 X. 협업 원칙 #4 (전략 제안 전 외부 리서치) 직접 위반.

## 8. 형용사 사후 라벨링 금지

**증상**: 디자인 결을 먼저 떠올린 후 형용사 사후 부여. 예: 잡지 결 → "차분/신뢰/클래식" 라벨링.

**원인**: 4축 만들고 싶어서 합집합에서 임의 픽 → 형용사로 라벨.

**가드레일**: **형용사 도출 → cluster화 → 결 결정** 순서 엄수. 결정한 결을 형용사로 정당화 X.

또 *어휘 nuance*가 아닌 *시각 결로 발현되는 것*으로 cluster 분류. 정리/절제/모던은 시각으로 95% 같음 — 같은 cluster.

## 9. 전략 vs 디자인 키워드 분리

**증상**: 제품 정체성 키워드 *겸손/배경/보조*를 시각에 직역. 결과: *자기 PR 약함* → 랜딩 본업과 충돌.

**원인**: 키워드를 한 묶음으로 처리. 전략(포지셔닝)과 디자인(시각 결) 구분 X.

**가드레일**: 키워드 도출 후 *전략 vs 디자인* 두 묶음으로 분류:

| 분류 | 의미 | 시각 직역 |
|---|---|---|
| 전략 | 포지셔닝/태도 | X (카피·톤만) |
| 디자인 | 시각 결로 발현 | O |

전략 키워드 시각 직역 = anti-pattern.

## 10. cliché 디테일 위험 — 비비드/Tactile/손글씨

**증상**: B 안 핫핑크+옐로+흑 3-color collision + stroke 외곽선 (광고 cliché). D 안 URGENT 스탬프 + 형광펜 직역 + generic Caveat 손글씨 (학생 다이어리 광고 cliché).

**원인**: 비비드/Tactile 결은 *세련된 표현*이 더 어려움. 직역하면 즉시 cliché.

**가드레일**:
- **손글씨/장식 = 수준 충족 OR 0**. 어중간 X. (금지 X — 잘 들어가면 OK)
- generic Google Fonts 손글씨(Caveat 등) 위험. 의도적 1군데, 제어된 사이즈만.
- Y2K 클리셰 (URGENT 스탬프, 형광펜 직역, 거대 % 배경, pill marquee clutter) 즉시 점검.
- 광고 톤(텍스트 stroke, 거친 외곽선) 회피.

## 11. 분기점 user 컨펌 의무 — 자율 진행 금지

**증상**: user "Phase 2, 3까지 하고 리뷰해보자" → Claude가 *전부 자율 진행 후 사후 리뷰*로 해석. 후보 1안 자체 선택 + 적용까지 끝낸 뒤 보고.

**원인**: anchor 역할 부재. user 발화의 표면(끝까지 가라)만 보고 의도(각 분기점 컨펌)를 놓침.

**가드레일**: 자율 micro와 컨펌 분기 분리.

| 자율 OK | 컨펌 필수 |
|---|---|
| 토큰 디테일 | Phase 진입/완료 |
| 섹션 body iteration | Phase 2 후보 1안 선택 |
| 미세 카피 | Phase 3 hero single message |
| 무드보드 작성 | 큰 구조 변경 |

→ 협업 원칙 #1 (anchor 역할).

## 12. 무드보드 ≠ mockup

**증상**: 색 후보 5안을 mood.html에 다크 base + 액센트 색만 변경해 시각화. user "이건 무드보드가 아니고 컬러 배리에이션이잖아".

**원인**: 무드보드 = 결 비교 도구인데 *우리 콘텐츠*로 시안 만들면 mockup 됨. 한 차원만 변형하면 컬러 배리에이션.

**가드레일**:
- 무드보드 = 외부 레퍼런스 콜라주 OR *결 다른* 가벼운 시안 비교
- 컬러 변수만 다른 건 무드보드 X
- 우리 콘텐츠로 시안 전체 만들면 mockup으로 변질

후보들은 모드/결/액센트 *세 차원*에서 의미 있게 갈라져야 함.

## 13. 레퍼런스 분석 패턴 — 공통 인사이트 추출

**증상**: user "카카오웹툰/Spotify Wrapped placeholder 뭐 들어가?" → Claude가 "카카오웹툰=일러스트, Spotify=데이터+typography" *분리만* 답.

**원인**: 분석 단계가 *분리 → 끝*. 공통 인사이트 추출 누락.

**가드레일**: 레퍼런스 분석 = 3단계:
1. **공통 인사이트 추출** — 둘 다 *임팩트 있는 콘텐츠 placeholder가 자기 자리를 차지*
2. **우리 제품 적용 옵션 변환** — (a) 학생 일상 영상 (b) 학습 데이터 시각화 (c) 컨셉 일러스트
3. **결정 시점 명시** — production에서 콘텐츠 확보 후

분리 나열만 하고 끝 = anti-pattern.
