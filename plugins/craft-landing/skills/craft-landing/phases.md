# Phases — 상세

## Phase 1 — 시작 prompt + 새 브랜치/페이지

### 입력
- `define.md` 또는 `prd.md` 또는 즉석 대화 — 제품 value, wow point, 가설, 타겟이 공유된 상태
- 최소 input. 디자인 가이드 X, 컬러 X, 톤 X.

### 시작 prompt (원본 그대로)

```
~한 랜딩을 만들거야.
어떠한 스킬도 사용하지 마. 너의 독창성을 최대한 뽐내고, 현대적인 감각으로, 모바일 퍼스트 랜딩 페이지를 그려줘.
이 세션에서는 지침이나 메모리가 하나도 없는 깔끔한 상태로 작업을 시작하고 싶어. 만약 가지고 있는 게 있어도 무시하고 바닐라 상태인 것처럼 작업해줘.
```

### 산출물
- draft 1 (HTML 한 파일). 이후 비판적 교정 대상.

### 보고 룰

draft 1 보고는 *결과만* X. 시각/내용 매 결정에 근거를 표 형태로 명시:

| 결정 | 분류 | 근거 |
|---|---|---|
| 다크 base + 라임 액센트 | 직관/차별화 | 학생 결 추정 (외부 리서치 X) |
| OS 메타포 | 직관 | 제품 정체성 (외부 검증 X) |
| 통계 1247명 | mock | placeholder |

분류는 (a) input 직접 (b) 레퍼런스 차용 (c) 직관/차별화 (d) 알면서 안고 간 risk 중 하나.

### 주의
- 이 단계에서 만든 결과를 *최종*으로 착각 금지. 이는 originality starting point일 뿐.
- 이후 phase에서 톤·내용·구조가 다 바뀔 수 있다.
- Phase 1 완료 → Phase 2 진입은 **user 컨펌 필수**. "Phase 2까지 해보자" 같은 발화도 *각 phase 분기점에서 컨펌* 받는다는 뜻.

## Phase 2 — 디자인 톤앤매너 교정

### 진입 전 외부 리서치 의무

다음 셋을 *외부 검색*으로 채운 후 진입. 회고/내 지식/추정으로 채우면 즉시 anti-pattern (#7).

1. **타겟 visual diet** — 와이즈앱, 한국언론진흥재단, KISDI, Korea Herald, 마케팅 인사이트 등 *통계 출처*. "토스/무신사" 같은 직관 추정은 검증 필요.
2. **글로벌 디자인 트렌드** — Creative Bloq, Fireart, Smashing 등. 2026 anti-AI / Gen Z fun and real / anti-minimalism (claymation, watercolor, tactile texture) 등 최신 흐름 확인.
3. **경쟁사 현황** — 같은 도메인 강자 + 다른 도메인 내러티브 강자 (Phase 1 draft에 끌려가지 말고 매트릭스로 다시).

### 흐름

1. **4-D 레퍼런스 매트릭스 채우기** (`references-matrix.md` 참조 + 외부 리서치 데이터)
2. **키워드 도출 — 전략 vs 디자인 분리**
3. **형용사화 → cluster화 → 결 결정**
4. **4안 매트릭스 — 전략적 분기 + 주축/부가**
5. **무드보드로 4안 시각화 — 결 차이를 보여줄 것** (컬러 변수만 다름 X)
6. **1안 선택** (user 컨펌 분기점)
7. **토큰화** — `tokens-schema.md`에 값 채움 (1안 결정 후, 후보 비교 단계 X)

### 1) 후보화 차원 분리

후보는 *세 차원*에서 각각 분기:

| 차원 | 예 |
|---|---|
| **모드** | 라이트 / 다크 |
| **결 (시각 인상)** | 미니멀 / 비비드 / Tactile / 클래식 / 게이미 |
| **액센트 색** | 라임 / 토스블루 / 라인그린 / 코랄 / 옐로 등 |

한 차원만 변형 = "컬러 배리에이션" (anti-pattern #12). 무드보드 X.

### 2) 키워드 도출 — 전략 vs 디자인 분리

| 분류 | 의미 | 예 |
|---|---|---|
| **전략 키워드** | 포지셔닝/태도. 카피·톤에는 영향, 시각 결에 직역 X | 겸손, 배경, 보조, 부담 적은, 곁에 있는 |
| **디자인 키워드** | 시각 결로 직역 가능 | 정리, 절제, 친밀, 가벼움, 해방감, 비비드, 활기 |

**전략 키워드를 시각에 직역하면 *자기 PR 약함* → 랜딩 본업과 충돌**. (예: "겸손/배경" 시각화 = 자기 표현 부족)

### 3) 형용사화 → cluster화 → 결 결정 (사후 라벨링 X)

**올바른 순서**:
1. 제품 정체성에서 형용사 도출 (~한, ~된)
2. 타겟 visual diet에서 형용사 도출
3. 두 셋을 *시각 결로 발현되는 것* 기준 cluster화 (어휘 nuance X — "정리/절제/모던"은 시각으로 95% 같음)
4. 진짜 교집합 vs 한쪽에만 있는 키워드 분리 (합집합을 교집합으로 부르지 말 것)
5. 결 결정

**금지**: 결을 먼저 떠올린 후 형용사 사후 부여 (anti-pattern #8).

### 4) 4안 매트릭스 — 전략적 분기 + 주축/부가

기본 분기 차원 = **전략적**:

| 안 | 분기 |
|---|---|
| A | 제품 정체성 우선 (주축) + 학생 친숙 (부가) |
| B | 학생 친숙 우선 (주축) + 제품 정체성 (부가) |
| C | 교집합만 |
| D | 양쪽 안 따름 — 새로운 안 (트렌드 기반 등) |

**핵심**: 교집합 자동 base는 안전하지만 4안 차별화 약화. *주축/부가 명시*로 의미 살림.

온도×분위기 같은 *시각 차원* 분기는 base 명확할 때만 보조로.

### 5) 무드보드 운용

- 무드보드 = *결 비교*가 목적. 외부 레퍼런스 콜라주 OR 같은 카피 위에 *결 다른* 가벼운 시안.
- 컬러 변수만 다른 건 무드보드 X (컬러 배리에이션).
- 우리 콘텐츠로 시안 전체 만들면 mockup이 됨 — 결 결정 방해.

### 핵심 룰

**색상/타이포/모티프/레이아웃 매 결정에 '이유'가 있어야 한다.**

거부할 정당화:
- ❌ "도파민 컬러여서"
- ❌ "Y2K 트렌드여서"
- ❌ "현대적이어서"
- ❌ "best practice"
- ❌ "AI 톤이라 보라/파랑" — 거짓 일반화 (Anthropic=코랄, OpenAI=그린, Vercel=흑백)

받아들일 정당화:
- ✅ "타겟 visual diet 데이터 (와이즈앱: 인스타 MAU 412만)에서 ___ 톤이 ___% 비중"
- ✅ "같은 도메인 X 사이트가 ___ 코드를 써서 카테고리 표준 / 차별화 의도"
- ✅ "다른 도메인 Y 사이트의 ___ 내러티브 차용 (왜: ___)"
- ✅ "2026 트렌드 출처 (Creative Bloq): anti-AI tactile 부상"

### AI 산출물 톤 회피

색의 문제 X. *결 전체*가 문제:
- 다크 + 추상 그라디언트 + glow
- minimal SaaS layout
- monospace 일변도
- 시스템 메시지 / OS 메타포 직접 노출
- 추상 data viz cliché

→ 다크 자체는 OK. 그라디언트/glow/SaaS 결을 피하면 됨.

### 산출물

- 디자인 톤 1안 (user 컨펌 후) + 토큰 셋 (색상·typography·spacing·radius 등)

## Phase 3 — 내용 (hero + 내러티브)

### 흐름

1. Hero에서 가장 전달할 *한 가지* 정의 (single message)
2. 랜딩 전반 내러티브 arc — 섹션 순서·전환·감정 흐름
3. 카피 N차 iteration
4. **어미·화자 톤 일관성 점검**

### Prompt 패턴

```
너는 이미 유명 에이전시의 시니어 마케터처럼 현대적·독창적으로 다시 고심해서 [내용/카피]을 잡아줘
```

### 핵심 룰 1 — 타겟 일상 언어로 번역

거부할 카피:
- ❌ PRD 내부 용어 그대로 (예: "관리 레이어", "메타인지 교정", "Constrained Autonomy")
- ❌ 제품 기능 나열 ("우리 앱은 X, Y, Z를 제공합니다")
- ❌ 타겟이 평소 안 쓰는 단어

받아들일 카피:
- ✅ 타겟이 매일 쓰는 어휘
- ✅ 박탈감/욕망/짜증 같은 감정 trigger
- ✅ 내부 용어 → 일상어 번역 ("메타인지 교정" → "뭘 빼야 하는지 아는 거야")

### 핵심 룰 2 — 어미·화자 톤 일관성

Hero 카피만 점검 X. 랜딩 *전반*의 어미/화자 톤 모니터링.

**흔한 실수** — 객관 OS 서술 + 친근 반말 혼재 → 기계 톤:
- "시스템이 학습합니다." (OS 객관) + "오늘 뭐 할까?" (친구 반말) ← 화자 일관성 깨짐

**default**:
- 화자 = **친구 권유** (제품 의인화 / OS 객관 서술 X)
- 어미 = 친근 반말 (~야, ~어, ~게 등). ~다 단정은 강조 1-2회만.
- 어미 분포 카운트 (`~다` N회, `~어` N회) 모니터링.

### 산출물

- Hero 카피 (single message) + 섹션별 내러티브 흐름 + 주요 카피
- user 컨펌 후 적용

## Phase 4 — 에셋 추가/수정

### v0 (TBD)

- 그래픽/애니메이션/일러스트
- **placeholder 먼저** 잡고, 실제 에셋 작업은 별도 단계
- 케이스 누적되면 세부화

### cliché 가드레일

- **손글씨/장식 = 수준 충족 OR 0**. 어중간하면 즉시 유치/cliché. (anti-pattern #10)
- generic Google Fonts 손글씨(Caveat 등) 위험. *의도적 1군데, 제어된 사이즈*가 안전.
- 비비드/Tactile 결은 *세련된 표현*이 더 어려움 — 절제/모던보다 cliché 위험 큼.
- 광고 클리셰 (스탬프, 형광펜 직역, 거대 % 배경) 즉시 점검.

## Cross-cutting — 모바일 분기

**처음부터 + 매 수정마다.**

- Phase 1 시작 prompt에 "모바일 퍼스트" 포함
- Phase 2 토큰화 시 desktop / mobile clamp range 동시 정의
- Phase 3 카피 길이 모바일 기준 검증
- Phase 4 에셋 비율·로딩 모바일 우선

데스크탑 먼저 짜고 모바일 맞추기 = anti-pattern.

## Cross-cutting — user 컨펌 분기점

자율 진행 가능: 토큰 디테일, 섹션 body, 미세 카피 iteration.

**user 컨펌 필수** (자율 X):
- Phase 진입/완료
- Phase 2 후보 1안 선택
- Phase 3 hero single message 결정
- 큰 구조 변경 (섹션 추가/삭제)

"Phase 2, 3까지 하고 리뷰해보자" 발화도 *각 분기점 컨펌* 의미. 자율 진행 후 사후 리뷰 X (anchor 역할 부재 = 협업 원칙 #1 위반).
