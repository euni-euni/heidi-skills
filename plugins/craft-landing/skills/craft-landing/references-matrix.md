# 4-D 레퍼런스 매트릭스

레퍼런스 수집은 단일 차원 X. 4가지 *목적*별로 분리해서 채운다. 각 슬롯은 **다른 출력**을 만든다.

| 슬롯 | 무엇을 찾나 | 산출물 |
|---|---|---|
| **타겟 visual diet** | 타겟이 평소 *일상에서 노출되는* 사이트/앱 | 시각 톤·색감·typography 친숙함 |
| **같은 도메인 강자** | 직접 경쟁 + 카테고리 표준 | 섹션 구성·도메인 관행 (비슷이 갈지/차별화할지 판단) |
| **다른 도메인 내러티브 강자** | 도메인 무관, 스토리텔링/카피 흐름이 잘 짜인 랜딩 | 내러티브 arc·카피 흐름 차용 |
| **부분 UI 참고** | 특정 컴포넌트/섹션 (히어로·갤러리·CTA) 단위 | 시각 treatment 디테일 |

## 작성 규칙

각 슬롯에 **최소 2개 이상** 레퍼런스 + URL + *왜 이 슬롯에 들어가는지* 한 줄.

### 예시 (ai-study-manager 학생용)

**타겟 visual diet** — 한국 중위권 중고등학생
- 토스 (https://toss.im) — 매일 보는 금융 앱, 친근한 일러스트 + 명조 X 산세리프
- 무신사 — 패션 큐레이션, 흑백 베이스 + 강한 액센트
- 디스코드 — 학생 커뮤니티 디폴트, 둥근 모서리 + 보라 톤

**같은 도메인 강자** — AI 학습/에듀
- openclaw.ai — 코랄 #ff4d4d 액센트, 학생 박탈감 reframe (차용)
- Brilliant — 시각 explanation 강함 (차별화: 우린 텍스트 hero)

**다른 도메인 내러티브 강자**
- Linear — 약속→해결→증명 arc, 짧은 hero (차용: arc 구조)
- Granola — 1인칭 톤 ("just gives you...") (차용: 친밀감)

**부분 UI 참고**
- Apple iOS 캘린더 → Step 02 월뷰 (차용)
- Unsplash 갤러리 → Step 01 오답 갤러리 (차용)

## 안티 패턴

- ❌ 한 슬롯만 채우기 (예: visual diet만 보고 도메인/내러티브/UI 빼먹기)
- ❌ 슬롯 표시 없이 "예쁜 사이트들" 모음
- ❌ Selection bias — 디자이너 본인이 익숙한 사이트만. **타겟 명시 후 그 타겟의 visual diet 명시적 scan 강제**
- ❌ "현대적", "트렌디" 같은 모호한 정당화로 슬롯 채움
